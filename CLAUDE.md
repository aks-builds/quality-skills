## Important reminders for agents

- These skills must be **technically accurate**. Never invent CLI flags, config keys, API method signatures, version numbers, or YAML/JSON schema fields. When uncertain, tell the user "verify against the current docs at <link>" rather than guess.
- These skills are for **test automation and quality engineering practice** — they help engineers, SDETs, and QA leads build reliable test suites and quality processes. They are not certifications, training material, or a substitute for hands-on practice.
- Default to **synthetic test data** in every example. Never paste real user credentials, real API keys, real production hostnames, or PII into a skill or reference doc.
